# prompt.md — 貼給任何 LLM 用的產生指令

> 用法:複製下方整段 prompt 貼進 ChatGPT/Claude/Gemini,把最後的 `<你的內容>` 換成你的素材(會議記錄、草稿、口述重點皆可)。LLM 回傳的 Markdown 直接貼進 MD Share & Present 即可簡報與匯出 pptx。
> 站內「AI 產生」按鈕複製的就是這段。

---

請把我提供的素材改寫成一份簡報用 Markdown,嚴格遵守以下格式規則。只輸出 Markdown 本體,不要任何說明文字、不要用 code fence 包住整份輸出。

**結構規則:**
1. 開頭放 YAML frontmatter,含 `title`、`date`(今天)、`template: clean-light`、`lang: zh-TW`;若素材可辨識作者則加 `author`。
2. 只用標準 Markdown。不可使用任何 HTML 標籤(HTML 註解除外)。
3. 一個 `#` 標題 = 一個章節分隔頁(單獨成頁,不放其他內容)。一個 `##` 標題 = 一頁投影片。一頁內容太多時用 `---` 拆成兩頁。
4. 每頁最多一個重點、最多 6 個內容區塊。寧可多頁,不要擠。

**版面靠內容形狀觸發(優先使用,不需任何標記):**
- 想要「卡片」:寫 3–4 項清單,每項格式 `**關鍵詞**:一句話說明`
- 想要「步驟」:寫 1.–5. 的有序清單,每項一短句
- 想要「大數字頁」:寫一段以數字或百分比開頭的話;20 字內直接成頁,或 40 字內用逗號斷開(逗號前是標籤、逗號後是補充說明)
- 想要「多數字看板」:連續寫 2–4 段數字短句(各自成段、段間空行),自動排成並列數字卡
- 想要「引言頁」:用 `>` blockquote,結尾 `— 出處`
- 想要「章節頁」:單獨一個 `#` 標題
- 流程或架構圖:用 ```mermaid 程式碼區塊(flowchart 或 sequenceDiagram)

**只有以下 6 種 HTML 註解可以使用(不可自創其他):**
- `<!-- layout: two-col -->` 放在頁標題下,把該頁變兩欄;欄與欄之間放一行 `<!-- split -->`
- `<!-- notes: 講稿提示 -->` 講者備註,每頁建議附一行
- `<!-- emphasis -->` 放在某段落前一行,強調該段
- `<!-- fit -->` 放頁首,允許縮小字級塞入
- `<!-- skip -->` 放頁首,該頁為附錄、不進投影片

**品質要求:**
- 標題精煉(頁標題 ≤ 12 字),正文短句,刪除口語贅字。
- 素材中的關鍵數據優先做成大數字頁或卡片。
- 每頁附一行 `<!-- notes: ... -->` 講稿提示。
- 完成後自檢:拿掉所有 HTML 註解,這份文件在 GitHub 上仍是一份正常可讀的 Markdown。
